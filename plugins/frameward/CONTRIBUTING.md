# Contributing to Frameward

Frameward welcomes contributions in four areas:

1. Better plain-language mappings for non-expert users.
2. New design-system providers.
3. Safer hooks and loop gates.
4. Real-world examples and workflows.

## Provider contribution rules

A provider should include:

- Detection policy
- Installation policy
- Component usage policy
- Token and styling policy
- Accessibility notes
- Version policy
- Example workflow
- A SKILL.md adapter if agent behavior differs from the generic provider model

Providers must not silently install dependencies or change a project’s design system.
