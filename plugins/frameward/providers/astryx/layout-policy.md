# Astryx Layout Policy

Astryx should be used frame-first.

## Process

1. Pick the shell.
2. Name regions.
3. Budget region sizes.
4. Choose row/table/card policy.
5. Define responsive behavior before filling content.

## Frameward alignment

This maps directly to Frameward’s layout region model.

## Anti-patterns

- Card soup: every record wrapped in a Card.
- Full-width Card stacks as page structure.
- Dense data rendered as decorative cards.
- Inspector panels that compress content instead of overlaying on narrow screens.
