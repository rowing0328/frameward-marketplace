# component-spec

## Goal

Specify component responsibility, states, props, variants, accessibility, and examples.

## When to use

Use before creating or heavily refactoring a component.

## Process

1. Name component responsibility.
2. Define props and variants.
3. Define states.
4. Define accessibility needs.
5. Map provider components if relevant.
6. Include examples and anti-patterns.
