# handoff-report

## Goal

Explain what changed, why it helps, and what was checked.

## When to use

Use at the end of Frameward work.

## Process

1. Summarize the original request.
2. Summarize the interpretation.
3. List changes in plain language.
4. List files changed.
5. Explain design-system/provider usage.
6. List QA checks.
7. Note assumptions and remaining risks.

## Output

```md
### What you asked for
-

### How I interpreted it
-

### What changed
-

### Why it helps
-

### Files changed
-

### Checks performed
-

### Assumptions
-

### Remaining risks
-
```

## User-language rule

Do not expose expert UI/UX terminology to non-expert users. Translate internal findings into plain language.
