# design-system-audit

## Goal

Prevent random UI changes by grounding work in existing project rules.

## When to use

Use before implementation for UI changes.

## Process

1. Inspect local components.
2. Inspect token/theme files.
3. Inspect styling conventions.
4. Find examples of similar screens.
5. Note constraints and anti-patterns.
6. Recommend safe primitives.

## Output

```md
### Existing UI rules
-

### Components to reuse
-

### Tokens or styles to reuse
-

### Anti-patterns to avoid
-

### Implementation notes
-
```

## User-language rule

Do not expose expert UI/UX terminology to non-expert users. Translate internal findings into plain language.
