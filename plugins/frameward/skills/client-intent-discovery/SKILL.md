# client-intent-discovery

## Goal

Translate vague client language into practical UI improvement intent.

## When to use

Use before implementation when the request is vague, subjective, or non-technical.

## Process

1. Capture the original request.
2. Identify likely intent categories.
3. Estimate screen type and risk.
4. Write assumptions in plain language.
5. Decide whether clarification is needed.
6. Ask at most one simple question for low/medium risk tasks.
7. For high-risk tasks, ask only essential plain-language questions.

## Output

```md
### Original request
-

### Plain-language interpretation
-

### Likely improvement direction
-

### Assumptions
-

### Risk level
-

### Clarifying question, only if needed
-

### Default action
-
```

## User-language rule

Do not expose expert UI/UX terminology to non-expert users. Translate internal findings into plain language.
