# screen-improvement-plan

## Goal

Create a small, safe implementation plan aligned with client intent.

## When to use

Use after intent, layout, provider, and design-system analysis.

## Process

1. Summarize the user’s request in plain language.
2. List safe changes.
3. List files likely to change.
4. Note provider constraints.
5. Define QA checks.
6. Keep scope small unless user asked for broad redesign.

## Output

```md
### Improvement direction
-

### Planned changes
-

### Files to inspect/change
-

### Provider rules
-

### QA plan
-

### Risks
-
```

## User-language rule

Do not expose expert UI/UX terminology to non-expert users. Translate internal findings into plain language.
