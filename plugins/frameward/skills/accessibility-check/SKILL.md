# accessibility-check

## Goal

Check keyboard, focus, labels, semantics, contrast, and state announcements.

## When to use

Use after implementation and before final handoff.

## Process

1. Check keyboard navigation.
2. Check visible focus.
3. Check labels and names.
4. Check semantic structure.
5. Check color contrast conceptually or with tools if available.
6. Check loading, empty, error, and disabled states.
7. Explain client-facing impact.
