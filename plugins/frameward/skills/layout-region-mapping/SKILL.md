# layout-region-mapping

## Goal

Map the screen into regions and responsibilities.

## When to use

Use before layout redesign, visual polish, responsive changes, or ratio analysis.

## Process

1. Identify screen type.
2. Name screen regions.
3. Mark each region as primary, secondary, utility, feedback, overlay, or safe area.
4. Define region behavior: fixed, sticky, scrollable, collapsible, overlay.
5. Check whether important content and actions are easy to find.
6. Write mobile transformation notes.

## User-language rule

Do not expose expert UI/UX terminology to non-expert users. Translate internal findings into plain language.
