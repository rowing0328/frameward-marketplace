# responsive-check

## Goal

Ensure screen priority survives across breakpoints.

## When to use

Use after implementation or before handoff.

## Process

1. Identify layout regions.
2. Define breakpoint behavior.
3. Check important button location.
4. Check touch target size.
5. Check overflow, tables, panels, and sticky areas.
6. Note unresolved risks.
