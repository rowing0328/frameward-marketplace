# visual-polish-pass

## Goal

Improve the screen without unnecessary redesign.

## When to use

Use for requests like cleaner, modern, polished, premium, or less crowded.

## Process

1. Confirm intent and assumptions.
2. Improve grouping and spacing.
3. Clarify what appears first.
4. Make the most important button easier to find.
5. Align styles with provider/design-system rules.
6. Check states and responsive behavior.
7. Explain changes in plain language.

```md
-

-
```

## User-language rule

Do not expose expert UI/UX terminology to non-expert users. Translate internal findings into plain language.
