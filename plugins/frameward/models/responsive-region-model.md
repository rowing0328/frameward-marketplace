# Responsive Region Model

Responsive behavior is about preserving task priority across screen sizes.

## Questions

- Which region stays visible?
- Which region moves below the main content?
- Which region collapses?
- Which region becomes an overlay?
- Does the important button remain easy to reach?
- Does the screen still explain what to do?

## Typical transformations

- Sidebar → drawer or bottom navigation
- Inspector panel → overlay or below-content section
- Two-column form → single-column flow
- Dense table → responsive table, list, or horizontal scroll with clear affordance
