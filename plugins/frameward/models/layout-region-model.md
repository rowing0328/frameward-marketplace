# Layout Region Model

Frameward starts from screen regions, not components.

## Common regions

- Global navigation
- Local navigation
- Page header
- Main content
- Supporting content
- Tool/filter area
- Important action area
- Feedback/status area
- Empty/loading/error state area
- Overlay area
- Footer
- Mobile safe area

## Region priorities

Each region should be classified as:

- Primary
- Secondary
- Tertiary
- Persistent
- Contextual
- Optional

## Region behaviors

- Fixed
- Sticky
- Scrollable
- Collapsible
- Overlay
- Conditional
- Responsive

## Main question

What should the user notice first, and what should they do next?
