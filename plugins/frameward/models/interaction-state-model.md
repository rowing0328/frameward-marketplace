# Interaction State Model

Frameward checks UI states because a screen is not complete in only its default state.

## Required states

- Default
- Loading
- Empty
- Error
- Disabled
- Hover
- Focus
- Pressed
- Success
- Partial data

## Client-facing explanation

Instead of saying “empty state,” say:

> I added a clearer message for when there is no content yet.

Instead of saying “focus state,” say:

> I made keyboard navigation easier to follow.
