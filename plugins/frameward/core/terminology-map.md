# Terminology Map

Frameward maintains two vocabularies.

## Internal vocabulary

Used by skills, hooks, schemas, and provider logic.

- CTA
- visual hierarchy
- layout region
- responsive contract
- token
- provider
- focus order
- state coverage
- affordance
- friction

## Client vocabulary

Used in summaries, questions, and handoff reports.

- the most important button
- what appears first
- main screen area
- mobile behavior
- existing style rules
- UI library
- keyboard movement
- cases like no data or errors
- looks clickable
- where the user gets stuck

## Rule

When in doubt, use client vocabulary.
