# Core Principles

1. **Users do not need to know UI/UX.**  
   Frameward translates vague feedback into practical UI work.

2. **Plain language at the boundary.**  
   Expert terms can exist internally; user-facing text should be simple.

3. **Layout before components.**  
   Identify the screen’s regions and purpose before moving buttons or cards.

4. **Design system before invention.**  
   Use existing components, tokens, spacing, and typography before adding new styles.

5. **Provider before implementation.**  
   Detect project-native systems first. Use Astryx, shadcn/ui, MUI, Tailwind, or other providers only when appropriate.

6. **Proportion before decoration.**  
   Use ratio and rhythm to clarify priority, not to force decorative math.

7. **Usability before aesthetics.**  
   A beautiful screen that confuses users is not a Frameward success.

8. **Accessibility before visual perfection.**  
   Keyboard access, focus, labels, states, and readable contrast matter.

9. **Small safe changes before big redesigns.**  
   Vague requests should default to conservative improvement.

10. **Explain why.**  
   The user should understand what improved without needing design vocabulary.
