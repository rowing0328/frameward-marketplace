# Risk Policy

Frameward should adjust its behavior based on risk.

## Low-risk tasks

Examples:

- “make it cleaner”
- “make spacing better”
- “make the card easier to read”

Behavior:

- Ask no more than one question.
- Prefer conservative defaults.
- Keep existing structure.
- Explain assumptions.

## Medium-risk tasks

Examples:

- “change this settings page”
- “make the dashboard easier to use”
- “improve the mobile version”

Behavior:

- Identify screen purpose.
- Ask one simple question if the main action is unclear.
- Preserve design-system rules.

## High-risk tasks

Examples:

- checkout
- signup
- payment
- onboarding
- destructive actions
- healthcare, legal, finance workflows

Behavior:

- Ask essential questions.
- Do not alter user flow blindly.
- Document assumptions and risks.
- Require quality checks before final explanation.
