# Client Language Policy

Frameward assumes the user does not know UI/UX terminology.

## Use plain phrases

| Internal term | Client-facing phrase |
|---|---|
| CTA | the most important button |
| Conversion | the action the user should complete |
| Visual hierarchy | what the user should notice first |
| Information architecture | how the screen content is organized |
| Cognitive load | how complicated the screen feels |
| Friction | where the user gets stuck |
| Empty state | what appears when there is no content yet |
| Error recovery | how the user can continue after something goes wrong |
| Design token | the project’s existing color, spacing, and text rules |
| Responsive QA | checking mobile, tablet, and desktop |

## Ask simple questions

Good:

```text
What should users do first on this screen?
A. Start using the product
B. Read the explanation
C. Compare options
D. Fill out the form
```

Bad:

```text
What is the primary CTA and conversion goal?
```

## Keep questions rare

For low-risk visual cleanup, make clear assumptions and proceed.

For high-risk flow changes, ask one to three simple questions.
